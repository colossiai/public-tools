# zh-CN/zh-TW vs  zh-Hans/zh-Hant


Ah, you’re asking about the difference between the two ways of labeling Chinese language/locale codes:

| 标记方式                  | 含义                                    | 用法/特点                                                                                                                       |
| --------------------- | ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **zh-CN / zh-TW**     | 基于**国家/地区**的标记。CN = 中国大陆，TW = 台湾      | ISO 3166 国家代码。主要用于地区化 (locale)，比如操作系统、浏览器、软件界面选择地区相关设置时用。它隐含了“语言 + 国家/地区”的意思，但不直接说明文字书写系统（简/繁体），虽然通常 zh-CN = 简体，zh-TW = 繁体。 |
| **zh-Hans / zh-Hant** | 基于**书写系统**的标记。Hans = 简体汉字，Hant = 繁体汉字 | ISO 15924 书写系统代码。更精确地表示文字形式，而不涉及地区。比如你可以在新加坡用 zh-Hans，也可以在香港用 zh-Hant，即便这些地区的国家代码不同。                                        |

✅ **关键区别**：

* zh-CN/zh-TW 强调地区，文字形式只是“通常默认的”。
* zh-Hans/zh-Hant 强调文字系统，不依赖地区。

举例：

* 台湾常用繁体中文 → zh-TW 或 zh-Hant
* 香港也用繁体中文 → zh-HK 或 zh-Hant
* 新加坡用简体中文 → zh-SG 或 zh-Hans

如果你想写**跨地区、跨平台的多语言支持**，推荐用 **zh-Hans / zh-Hant**，因为它明确区分文字形态，而不是混淆地区。
