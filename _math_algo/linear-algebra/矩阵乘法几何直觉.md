# 矩阵相乘 AB 可以认为是在一个单位向量上做连续变换吗？先 p=Bu, 然后 result=Ap


对，**这是一个非常正确而且很“线代几何化”的理解**。但要补一个小修正：

> 不一定是“单位向量（unit vector）”，而是**任意向量** (u)。
> 如果取单位向量，特别容易看清矩阵在“扭曲空间”时做了什么。

你写的：

```math
p = Bu
```

然后：

```math
result = Ap
```

合起来：

```math
result = A(Bu)
```

也就是：

```math
result = (AB)u
```

这正是矩阵乘法的本质：

> **AB = 先做 B 变换，再做 A 变换**

---

## 几何直觉：矩阵 = 空间变换器

把向量 (u) 想象成空间里的一根箭头。

1. 先经过 (B)

空间被拉伸/旋转/压缩：

```math
u \rightarrow Bu
```

得到新箭头 (p)。

2. 再经过 (A)

继续变换：

```math
p \rightarrow Ap
```

最终：

```math
u \rightarrow ABu
```

所以：

> **矩阵乘法就是空间变换的组合（composition of transformations）**

顺序特别重要：

```math
ABu
```

意味着：

* **先 B**
* **后 A**

因为矩阵离向量最近的先执行。

像函数：

```math
f(g(x))
```

先 (g)，后 (f)。

---

## 为什么“单位向量”特别有帮助？

如果取标准基向量（basis vectors）：

```math
e_1=
\begin{bmatrix}
1\\
0
\end{bmatrix}
,\quad

e_2=
\begin{bmatrix}
0\\
1
\end{bmatrix}
```

它们是长度为 1 的单位向量。

观察：

```math
Ae_1
```

和

```math
Ae_2
```

你会发现：

> **矩阵其实就是在描述“坐标轴被送去了哪里”**

例如：

```math
A=
\begin{bmatrix}
2 & 1\
0 & 3
\end{bmatrix}
```

则：

```math
Ae_1=
\begin{bmatrix}
2\\
0
\end{bmatrix}
```

```math
Ae_2=
\begin{bmatrix}
1\\
3
\end{bmatrix}
```

说明：

* x轴单位方向被送到 ((2,0))
* y轴单位方向被送到 ((1,3))

整个空间就跟着变形。

所以从这个角度：

> **矩阵 = 对单位基向量做了什么**

而：

> **矩阵乘法 AB = 连续做两次空间变换**

---

甚至可以把你写的理解升级成一句非常强的线代直觉：

> **矩阵乘法不是数字计算，而是“变换管道”**

```text
u
↓ B
Bu
↓ A
ABu
```

这套直觉以后直接通到：

* Transformer 里的 (Q=WX)
* attention 的线性投影
* 多层神经网络连续变换
* 特征向量（连续变换后方向不变）

因为神经网络本质就是：

```text
x
↓ W1
↓ nonlinearity
↓ W2
↓ nonlinearity
...
```

即：

> **不断在高维空间里做连续变换。**
