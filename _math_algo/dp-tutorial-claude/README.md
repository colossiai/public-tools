# 动态规划入门教程（零基础友好）

一份**用 Python 写、用图说话**的动态规划入门教程。每个知识点一个独立可运行的脚本，跑起来就出图。

---

## 这份教程是给谁的

- 你写过几行代码，但**从没接触过动态规划（DP）**。
- 你看过别人讲 DP，但**总觉得"为什么这么定义状态"很玄学**。
- 你不想只背模板，**想看到 DP 是怎么一步步从暴力解推导出来的**。

如果以上三条命中任意一条，这份教程就是为你写的。

---

## 学习路径

按编号顺序读，每个文件 15–30 分钟。前 5 个文件建立"DP 是什么、怎么写"的基础，后 7 个文件覆盖最常考的 7 种模式。

### 第一部分：DP 的基础思维

| # | 文件 | 学完你会知道 |
| -- | ---- | ------------ |
| 01 | `01_why_dp_recursion_tree.py` | 为什么朴素递归慢成那样 — 看见**重复子问题** |
| 02 | `02_memoization_top_down.py` | 把算过的答案记下来 — **记忆化（自顶向下）** |
| 03 | `03_tabulation_bottom_up.py` | 反过来从小往大算 — **表格法（自底向上）** |
| 04 | `04_rolling_array.py` | 大部分时候你只需要"最近两格"— **滚动数组优化** |
| 05 | `05_dp_design_template.py` | 拿到一道新题，怎么从零想清楚 — **DP 四步法** |

### 第二部分：七种最常见的 DP 模式

| # | 文件 | 模式 | 代表题型 |
| -- | ---- | ---- | -------- |
| 06 | `06_coin_change_linear.py` | 线性 DP | 凑硬币、爬楼梯、抢房子 |
| 07 | `07_unique_paths_grid.py` | 网格 DP | 不同路径数、最小路径和 |
| 08 | `08_knapsack_01.py` | 背包 DP | 0/1 背包、子集和、分割等和 |
| 09 | `09_lis_sequence.py` | 序列 DP | 最长上升子序列、最长公共子序列 |
| 10 | `10_edit_distance.py` | 双序列 DP | 编辑距离、字符串匹配 |
| 11 | `11_matrix_chain_interval.py` | 区间 DP | 矩阵链乘、戳气球、回文分割 |
| 12 | `12_tsp_bitmask.py` | 状态压缩 DP | 旅行商、集合覆盖（小规模） |

## 运行方式

需要先装 [`uv`](https://docs.astral.sh/uv/)（推荐）：

```bash
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

然后：

```bash
cd _math_algo/dp-tutorial-claude

# 安装依赖（自动创建 .venv）
uv sync

# 跑任意一个文件——默认只打印讲解，不出图
uv run python 01_why_dp_recursion_tree.py

# 想看图？加 --plot（弹窗）或 --save（写到 ./plots/）
uv run python 08_knapsack_01.py --plot
uv run python 08_knapsack_01.py --save
uv run python 08_knapsack_01.py --plot --save     # 两者一起
```

### CLI 开关

每个章节脚本都支持下面两个开关，**都是 opt-in（默认关闭）**：

| 开关      | 行为                                            |
| --------- | ----------------------------------------------- |
| `--plot`  | 弹出 matplotlib 窗口显示图                       |
| `--save`  | 把图保存到 `./plots/<编号>_<名称>.png`           |
| 都不加    | 只在终端打印讲解 / DP 表 / 对拍结果，不动图      |

这样设计是因为：

- 在终端/CI 里读讲解，不需要每次都弹窗；
- 想留作笔记时再用 `--save`；
- 课堂演示用 `--plot --save` 两个一起开。

## 怎么读每个文件

每个 `.py` 文件顶部都有一个大 docstring，固定三段结构：

- **【思维】** — 不写代码，先讲"如果我什么都不知道，会怎么暴力做？为什么这种做法不行？怎么自然地想到 DP？"
- **【方案】** — 落地步骤：状态怎么定义、转移方程怎么写、边界是什么、计算顺序怎么定。
- **【可视化】** — 跑出来的图说明了什么。

**强烈建议**：先**读完顶部 docstring**，再看代码。代码只是 docstring 思维的实现。

## 一句话记住 DP

> **DP = 把小问题的答案记下来，大问题查表就行。**
>
> 难的从来不是写代码，难的是**怎么定义"小问题"**。这份教程的 12 个文件，本质上就在讲这一件事的 12 种练习。
