# LeetCode 系统性解题方法论

> 目标：把「刷题」变成「训练模式识别 + 建立可复用模板」，而不是靠题海记忆。

---

## 一、总体心法

刷 LeetCode 的本质不是记住每道题，而是：

1. **模式识别**：看到题目 → 快速映射到某一类「套路」。
2. **模板复用**：每一类套路都有一个可背诵的骨架代码。
3. **约束推断复杂度**：从数据规模反推该用什么复杂度的算法。
4. **刻意练习**：错题重做、限时训练、讲解复盘。

> 一句话：**从 constraints 猜复杂度，从复杂度猜算法，从算法套模板。**

---

## 二、五步解题框架（UMPIC）

面试与练习都适用的标准流程：

| 步骤 | 名称                 | 关键动作                                        |
| -- | ------------------ | ------------------------------------------- |
| U  | Understand（理解）     | 复述题意、明确输入输出、边界与返回值、能否修改输入                   |
| M  | Match（匹配）          | 归类到已知模式（见第四节），回忆对应模板                         |
| P  | Plan（规划）           | 先写伪代码/思路，确认复杂度是否满足 constraints               |
| I  | Implement（实现）      | 写干净代码，变量命名清晰，先正确再优化                          |
| C  | Check / Complexity | 手跑样例 + 边界用例，说清时间/空间复杂度                       |

**沟通提示**（面试）：先讲暴力解 → 分析瓶颈 → 给出优化解，而不是直接憋最优解。

## 三、从数据规模反推算法

看到 constraints 里的 `n` 上限，几乎能锁定目标复杂度：

| n 的规模        | 允许的复杂度              | 常见算法                            |
| ------------ | ------------------- | ------------------------------- |
| n ≤ 10~12    | O(n!)、O(2ⁿ · n)     | 全排列、暴力回溯、状态压缩 DP                |
| n ≤ 20       | O(2ⁿ)               | 子集枚举、bitmask                    |
| n ≤ 100      | O(n³)               | Floyd、区间 DP                     |
| n ≤ 1,000    | O(n²)               | 双重循环、朴素 DP                       |
| n ≤ 10⁵~10⁶  | O(n log n)、O(n)     | 排序、二分、双指针、滑窗、单调栈、哈希、堆           |
| n ≥ 10⁷~10⁸  | O(n)、O(log n)、O(1)  | 线性扫描、数学公式、位运算                    |

> 经验值：1 秒约能跑 **10⁸** 次基本运算。

## 四、核心模式与模板速查

按「看到什么 → 想到什么」组织：

### 1. 双指针 / 对撞指针
- **信号**：有序数组、两数之和、回文、去重、原地移除。
- **要点**：`left/right` 向中间收缩，或快慢指针。
```text
left=0, right=n-1
while left < right:
    根据条件移动 left 或 right
```

### 2. 滑动窗口
- **信号**：连续子数组/子串、最长/最短、「不超过 K 个」。
- **要点**：右指针扩张，违反条件时左指针收缩。
```text
for right in range(n):
    加入 s[right]
    while 窗口不合法:
        移出 s[left]; left += 1
    更新答案
```

### 3. 二分查找 / 二分答案
- **信号**：有序、「最小的最大值 / 最大的最小值」、单调可行性。
- **要点**：确定 `check(mid)` 单调性，套模板避免死循环。
```text
lo, hi = 边界
while lo < hi:
    mid = (lo + hi) // 2
    if check(mid): hi = mid
    else: lo = mid + 1
return lo
```

### 4. 哈希表
- **信号**：查重、计数、配对（两数之和）、前缀和 + 目标差。
- **要点**：空间换时间，O(1) 查找。

### 5. 前缀和 / 差分
- **信号**：区间和、子数组和为 K、频繁区间修改。
- **要点**：`prefix[i] = prefix[i-1] + a[i]`；差分做区间批量更新。

### 6. 单调栈 / 单调队列
- **信号**：下一个更大/更小元素、柱状图最大矩形、滑窗最值。
- **要点**：栈内保持单调，弹出时结算答案。

### 7. 栈 / 队列
- **信号**：括号匹配、表达式求值、BFS 层序。

### 8. 链表
- **信号**：反转、环检测、合并、找中点。
- **要点**：dummy 头节点、快慢指针。

### 9. 树 / 二叉树
- **信号**：遍历、路径、最近公共祖先、BST 性质。
- **要点**：递归三步——终止条件、单层逻辑、返回值；DFS/BFS 二选一。

### 10. 图（BFS / DFS / 并查集 / 拓扑排序）
- **信号**：连通性、最短路（无权用 BFS）、依赖顺序、岛屿。
- **要点**：visited 去重；拓扑排序用入度；连通分量用并查集。

### 11. 回溯
- **信号**：全排列、组合、子集、棋盘、切割。
- **要点**：选择 → 递归 → 撤销选择。
```text
def backtrack(path, choices):
    if 满足终止: 记录答案; return
    for c in choices:
        做选择
        backtrack(path+[c], 剩余)
        撤销选择
```

### 12. 动态规划
- **信号**：最优子结构 + 重叠子问题；最值/计数/可行性。
- **五步**：定义状态 → 状态转移方程 → 初始化 → 遍历顺序 → 返回值。
- **子类**：线性 DP、背包、区间 DP、树形 DP、状态压缩 DP、数位 DP。

### 13. 贪心
- **信号**：局部最优可推出全局最优（需能证明或反例验证）。
- **要点**：常配合排序；不确定时先用 DP 保底。

### 14. 堆 / 优先队列
- **信号**：Top-K、第 K 大、合并 K 个有序、动态中位数（双堆）。

### 15. 位运算 / 数学
- **信号**：只出现一次、集合枚举、GCD、快速幂、质数筛。

## 五、复杂度分析清单

- 说清 **时间** 与 **空间** 复杂度，包含递归栈开销。
- 注意隐藏成本：字符串拼接、切片、`in` 对 list 是 O(n)。
- 递归复杂度用「递归树」或「主定理」估算。

## 六、按主题的推荐学习路径

建议按 **主题成块攻克**，而非随机刷：

1. 数组 & 字符串（双指针、滑窗、前缀和）
2. 哈希表
3. 栈 & 队列（含单调栈）
4. 链表
5. 二分查找
6. 树 & BST
7. 图（BFS/DFS/并查集/拓扑）
8. 回溯
9. 动态规划（最难，放最后，单独深挖）
10. 堆 / 贪心 / 位运算 / 数学

## 七、刻意练习方法

| 方法       | 做法                                        |
| -------- | ----------------------------------------- |
| 限时训练     | Easy 15 min / Medium 30 min / Hard 45 min |
| 卡住就看题解   | 卡 20~30 分钟无思路 → 看解法，**理解后合上重写**           |
| 错题本      | 记录：题号、模式、卡点、正确思路一句话总结                      |
| 间隔重做     | 1 天 / 1 周 / 1 月后重做，直到肌肉记忆                  |
| 讲解复盘     | 用「费曼技巧」把解法讲给别人（或对空气讲）                      |
| 手写 & 讲思路 | 面试常无 IDE，练习白板/口述先讲思路再编码                    |

## 八、常见陷阱清单

- 边界：空输入、单元素、全相同、最大/最小值溢出。
- 下标越界、`while` 死循环（二分边界写错）。
- 修改容器时同时遍历。
- 整数溢出（其他语言）与负数取模。
- 递归深度过大（改迭代 / 增大栈）。
- 没读清返回要求（返回下标 vs 值、是否去重、是否需要所有解）。

---

## 九、面试沟通模板

1. **澄清**：确认输入范围、是否有重复、是否排序、如何处理非法输入。
2. **举例**：手动走一个小样例，确认理解一致。
3. **给思路**：先暴力，再指出瓶颈与优化方向。
4. **报复杂度**：编码前说清预期时间/空间复杂度。
5. **编码**：边写边解释关键决策。
6. **验证**：主动跑样例 + 边界用例，主动指出可优化点。

## 十、一页速记

```text
读题 → 看 constraints 猜复杂度 → 匹配模式 → 套模板 → 分析边界 → 报复杂度
```

**模式触发词速记**：
- 连续子数组/子串 → 滑动窗口
- 有序 + 找目标/边界 → 二分
- 两数配对/查重/计数 → 哈希
- 下一个更大/更小 → 单调栈
- 区间和 → 前缀和
- 所有组合/排列/子集 → 回溯
- 最值/计数 + 重叠子问题 → 动态规划
- Top-K / 第 K 大 → 堆
- 连通性/最短路(无权)/依赖顺序 → 图 (BFS/DFS/并查集/拓扑)
