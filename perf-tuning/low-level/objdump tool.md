`what is objdump`

`objdump` is a command-line tool used to **inspect compiled binary files** (object files, executables, shared libraries). It is part of the **GNU Binutils** suite and is commonly used by systems programmers, reverse engineers, and low-latency developers.

It essentially lets you **“look inside” compiled machine code**.

---

# 1. What `objdump` is used for

You use `objdump` to display:

| Feature       | What it shows                     |
| ------------- | --------------------------------- |
| Assembly code | Disassembled machine instructions |
| Symbol table  | Functions and variables           |
| Sections      | `.text`, `.data`, `.bss`, etc     |
| Headers       | Binary metadata                   |
| Relocations   | Linker relocation entries         |
| Debug info    | If compiled with `-g`             |

Typical files it analyzes:

* `.o` (object files)
* executables
* `.so` shared libraries
* static libraries `.a`

Example:

```bash
objdump -d program
```

This **disassembles the executable into assembly code**.

---

# 2. Example

### C program

```c
int add(int a, int b) {
    return a + b;
}
```

Compile:

```bash
gcc -O2 -c add.c
```

Now inspect:

```bash
objdump -d add.o
```

Output (simplified):

```
0000000000000000 <add>:
   0: 89 f8     mov %edi,%eax
   2: 01 f0     add %esi,%eax
   4: c3        ret
```

Meaning:

| Instruction     | Meaning         |
| --------------- | --------------- |
| `mov %edi,%eax` | move argument a |
| `add %esi,%eax` | add argument b  |
| `ret`           | return          |

So `objdump` reveals the **actual CPU instructions generated by the compiler**.

---

# 3. Most useful `objdump` options

### Disassemble code

```bash
objdump -d program
```

---

### Disassemble with source code

```bash
objdump -S program
```

Shows **C code + assembly side-by-side** (if compiled with `-g`).

---

### Show symbol table

```bash
objdump -t program
```

Example:

```
0000000000001130 g     F .text 0000000000000020 main
```

Shows functions and addresses.

---

### Show sections

```bash
objdump -h program
```

Example:

```
Idx Name      Size
0   .text     00000234
1   .data     00000020
2   .bss      00000080
```

---

### Intel syntax (much nicer)

```bash
objdump -d -M intel program
```

Example:

```
mov eax, edi
add eax, esi
ret
```

# 4. Why systems programmers use it

`objdump` is extremely useful for:

### 1️⃣ Performance engineering

See how compiler optimized code.

Example:

```cpp
if (x) y++;
```

You can check if compiler produced:

```
branch
```

or

```
cmov
```

(important for **branch prediction optimization**).

---

### 2️⃣ Low-latency / HFT development

You verify:

* instruction count
* branch instructions
* memory loads
* vector instructions (AVX/SSE)

---

### 3️⃣ Reverse engineering

You can inspect unknown binaries.

Example:

```bash
objdump -d malware
```

---

### 4️⃣ Debugging compiler behavior

Sometimes compilers generate unexpected code.

`objdump` lets you see **exact machine instructions**.

---

# 5. Relationship with other binary tools

| Tool      | Purpose              |
| --------- | -------------------- |
| `objdump` | inspect binaries     |
| `nm`      | list symbols         |
| `readelf` | inspect ELF metadata |
| `strings` | extract strings      |
| `gdb`     | debug binaries       |

All come from **GNU Binutils**.

---

# 6. Very useful command combo

Many low-level developers use:

```bash
g++ -O3 -S code.cpp
```

to generate assembly.

Or:

```bash
objdump -d -M intel program
```

to inspect compiled binary.

---

✅ **Rule of thumb**

> If you want to know *what the CPU actually executes*, use `objdump`.
